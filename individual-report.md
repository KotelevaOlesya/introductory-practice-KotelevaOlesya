6. Рефлексия
Что было интересным:
1.	Работа в команде - первый опыт реальной коллаборации над проектом был очень ценным. Особенно интересно было участвовать в Code Review и видеть разные подходы к решению задач.
2.	Git и GitHub - осознал мощь этих инструментов для контроля версий и совместной работы. Стало понятно, почему это стандарт в индустрии.
3.	Документирование - открыл для себя, что хорошая документация не менее важна, чем работающий код. Markdown оказался простым и эффективным инструментом.
4.	Практическое применение - возможность сразу применять изученное на реальном проекте сильно мотивировало.
Что вызвало трудности:
1.	Конфликты слияния - первый конфликт в Git был стрессовым, но после изучения механизмов разрешения стало понятнее.
2.	Тайм-менеджмент - балансировать между индивидуальными заданиями и групповой работой требовало планирования.
3.	Code Review - сложно было давать конструктивную обратную связь, не обидев коллег. Научился формулировать комментарии в позитивном ключе.
4.	Настройка окружения - различные проблемы с зависимостями и версиями ПО на разных компьютерах участников.
Чему научились:
1.	Технические навыки:
o	Полный цикл работы с Git: от коммита до Pull Request
o	Эффективное использование VS Code с расширениями
o	Создание технической документации на Markdown
o	Основы командной разработки
2.	Мягкие навыки (soft skills):
o	Коммуникация в технической команде
o	Распределение задач и ответственности
o	Давать и принимать обратную связь
o	Презентация результатов работы
3.	Процессы:
o	Работа по agile-методологии (ежедневные стендапы)
